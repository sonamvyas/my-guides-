Foundations & Setup

Week 1 - 2

What is Tosca?

  • Model-based test automation tool by Tricentis
  • No-code / low-code approach
  • Supports web, desktop, mobile, API, SAP
  • Integrated with CI/CD pipelines
  • Used by enterprise QA teams globally

Tosca vs Other Tools

  • vs Selenium: No scripting needed in Tosca
  • vs UFT/QTP: Better model-based reuse
  • vs Cypress: Tosca covers more app types
  • vs Manual: Much faster regression cycles
  • Best for: Enterprise + SAP + Salesforce QA

Core Concepts

  • Modules — scanned UI element definitions
  • TestCases — sequence of test steps
  • TestSuites — grouped test cases
  • Execution Lists — run configurations
  • Workspace — your project container

Setup Checklist

💡 Tip: Use the Tricentis demo app "Demowebshop" (available for free) to practice all scanning and test building exercises.

Core Skills — Scanning & Test Cases

Week 3 - 4

Scanning Applications

  • Open TBox Scan to scan web elements
  • Scan buttons, fields, dropdowns, tables
  • Review and clean up scanned modules
  • Understand steering properties
  • Re-scan when UI changes
// TBox Scan Steps
1. Open Tosca Commander
2. Right-click Modules folder
3. Select "Scan Application"
4. Highlight the UI element
5. Click "Scan Selected"

Building Test Cases

  • Drag modules into a TestCase
  • Add TestSteps from module attributes
  • Set ActionMode for each step
  • Add expected values for verification
  • Run and observe results

ActionMode Types

  • Input — type text into a field
  • Verify — assert a value is correct
  • Select — pick from a dropdown
  • Click — click a button/link
  • WaitOn — wait for element to appear

Test Data & Parameters

Week 5 - 6

TestCase Design (TCD)

  • Separate test logic from test data
  • Create equivalence classes
  • Define representative values
  • Auto-generate test case combinations
  • Reduce manual data management

Buffers

  • Store runtime values between steps
  • Use {BUFFER[name]} syntax to reference
  • Read values from UI and store them
  • Pass data across TestSteps
  • Great for dynamic IDs and timestamps
// Buffer example
ActionMode: ReadBuffer
Buffer Name: OrderID
Value: {BUFFER[OrderID]}

Business Parameters

  • Reusable parameters across test cases
  • Centralize environment-specific values
  • URL, username, password management
  • Switch between test environments easily
  • Stored in Parameter Sets

Stage 3 Checklist

⚠️ Common Mistake: Don't hardcode URLs or credentials in TestSteps. Always use Business Parameters so tests are environment-portable.

Advanced Features

Week 7 - 8

Libraries (Reusable TestCases)

  • Create reusable login / logout flows
  • Call library steps from any TestCase
  • Parameters make them flexible
  • Reduces duplication across test suites
  • Update once, fix everywhere

Recovery Scenarios

  • Handle unexpected popups/dialogs
  • Auto-dismiss cookie banners
  • Recover from timeout errors
  • Keep tests running in CI pipelines
  • Define cleanup steps on failure

Conditions & Logic

  • If/Else branching in test steps
  • Skip steps based on conditions
  • Loop over repeating actions
  • Combine with buffers for dynamic flows
  • Validate optional UI elements

API Testing

Week 9 - 10

API Test Setup

  • Use Tosca API Scan to import endpoints
  • Supports REST and SOAP APIs
  • Import from Swagger/OpenAPI specs
  • Set headers, auth tokens, base URLs
  • Parameterize requests with buffers
// REST request example
Method: POST
Endpoint: /api/v1/accounts
Body: {"name": {BP[AccName]}}
Verify: status == 201

Response Verification

  • Verify HTTP status codes
  • Extract JSON response values
  • Store response data in buffers
  • Chain API calls (get ID, then use it)
  • Validate response schema

UI + API Combined

  • Create a record via API, verify in UI
  • Submit a form in UI, verify via API
  • Use API for test data setup/teardown
  • Faster and more stable than UI-only tests
  • Ideal for Salesforce end-to-end flows

Execution, CI/CD & Reporting

Week 11 - 12

Execution Lists

  • Organize TestCases into run lists
  • Set execution order and grouping
  • Configure parallel execution agents
  • Schedule automated runs
  • Filter by tags or attributes

CI/CD Integration

  • Tosca CI plugin for Jenkins
  • Azure DevOps task available
  • Trigger runs via command line (TCShell)
  • Pass results back to CI pipeline
  • Gate deployments on test pass rate
// TCShell command
TCShell.exe
 -workspace "C:\Tosca\WS"
 -executionlist "Regression"
 -result "results.xml"

Reporting & Logs

  • View pass/fail in Execution Log
  • Screenshot capture on failure
  • Export reports to PDF/HTML
  • Tosca Dashboard for trends
  • Integrate with Jira for defects
